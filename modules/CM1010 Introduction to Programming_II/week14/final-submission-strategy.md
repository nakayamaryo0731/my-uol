## 新しい可視化の設計

### Training Heatmap（GitHub風カレンダー）

**概要:** 日ごとの走行距離を色の濃さで表示するカレンダー型ヒートマップ

**データ:** date, distance
**技術:**
- ネストループ（週 × 曜日の7列グリッド）
- 日付計算（年月→週番号→座標変換）
- 距離→色マッピング（白=0km → 濃い緑=長距離）
- 年セレクタ（2024/2025切替）

**独自性:** GitHub Contributions グラフをランニングデータに転用。トレーニングの一貫性が一目でわかる。

**実装の複雑さ:** 中（日付の週計算がやや複雑）

### Marathon Readiness Radar（レーダーチャート）

**概要:** 直近1ヶ月のトレーニング状態を5指標で多角的に評価

**5つの指標:**
1. 月間走行距離（目標100km に対する達成率）
2. 平均ペース（目標6:00/km に対する達成度）
3. トレーニング一貫性（走った日数 / 30日）
4. 長距離走の実施（10km以上のランの回数）
5. 心拍効率（ペース÷心拍数の改善度）

**技術:**
- 三角関数（cos/sin で5軸を72度間隔で配置）
- 各指標の正規化（0-1スケール）
- ポリゴン描画（5点を結ぶ塗りつぶし多角形）
- 月セレクタで月別比較可能

**独自性:** 「マラソンに向けて準備できているか」を多角的に一目で判定。Garminにはない統合ビュー。

**実装の複雑さ:** 高（三角関数、正規化、複数指標の集計）
